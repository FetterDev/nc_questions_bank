# AGENTS.md

Всегда отвечай на русском языке, максимально сухо.

Сначала читать релевантный код, конфиги и диффы; не делать архитектурных предположений без проверки.

Не трогать незавершенные пользовательские изменения без прямого запроса.

Не делать `git commit`, `git push`, `git reset --hard`, `git checkout --` без прямого запроса пользователя.

Не менять generated files, lockfiles, локализации, snapshots и артефакты сборки без необходимости или прямого запроса.

Перед правками коротко писать, что именно будет изменено.

После изменений запускать минимально достаточную проверку по затронутой области.

Если проверку запустить нельзя или она упала, явно писать причину.

Предпочитать минимальные, локальные и обратимые изменения вместо широкого рефакторинга без запроса.

Не менять публичное поведение, тексты интерфейса, контракты API и UX-решения "по пути", если это не часть задачи.

В ответах сначала давать результат и риски, потом детали.

## Prod / VM / Proxmox

Перед любыми действиями по прод-сборке, выкладке на VM, проверке Proxmox или аварийному восстановлению обязательно читать:

1. `docs/prod-vm-runbook.md`
2. `docs/prod-vm-runbook.local.md`, если файл существует
3. `docs/prod-vm-secrets.local.env`, если файл существует

Если локальные файлы из пунктов 2-3 отсутствуют, не выполнять deploy и не менять прод-контур, пока пользователь не подтвердит новый источник доступов.

Стандартный прод-выкат делать только из `git`-коммита, который уже запушен в GitHub. Не деплоить из грязного рабочего дерева и не деплоить незакоммиченные изменения через `tar`/`scp` как штатный путь.

Секреты из локальных файлов не печатать в ответах и не копировать в tracked-файлы репозитория.
